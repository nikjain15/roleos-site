RoleOS · Live

I built RoleOS — an operating system to run my own job search.

Most candidates read JDs one at a time. I shipped a small system that scopes companies, fetches postings, and uses Claude to extract every requirement into a strict schema — so I can rank fit, spot gaps in my own profile, and prioritize where to apply. This page tracks it live, end-to-end. The point isn't the job search — it's how I leverage AI to compress weeks of manual work into hours.

Architecture — four scripts, one schema

01 · SCAN
Discover roles
Crawl ATS portals (Greenhouse, Lever, Ashby) for ~70 target companies. Filter to PM, AI, and growth roles.
02 · FETCH
Raw JD capture
Pull each posting to markdown, snapshotted with URL + provider + job ID for reproducibility.
03 · EXTRACT
Claude → strict JSON
Per-JD LLM call with a constrained schema: archetype, seniority, must-haves with verbatim quotes, salary, location, visa. Zero hallucination by contract.
04 · RENDER
Candidate-facing view
Compile structured JSONs into readable markdown digests for fast scanning and fit assessment.
