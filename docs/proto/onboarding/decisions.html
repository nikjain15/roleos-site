Onboarding — design decisions

Status: APPROVED (2026-07-04) · prototype at /proto/onboarding/ · formal PRD shipped: roleos-app PR #37 (docs/specs/onboarding-design.md + J1 board slice) — human merges, builder loop builds.

Confirmed intent

Convert a skeptical senior candidate from “another AI job tool” to “an agent just did real work for me” in under a minute, and fire the first taste-model signals before signup. One input (LinkedIn / paste / CV, plus an optional one-line target) → honest plain-English ticker → one combined screen: her tappable read of you (statements + guess + one insight, as bold as evidence allows) beside jobs that stream in and visibly re-rank when you correct her → a save-framed signup with a real taste of the work → a first feed minute where RO narrates next moves, asks for a goal, matches beneath. Every dead end gets an honest scripted recovery — she never fakes value. Success = mirror reached in ~30s of effort, ≥1 reaction, work saved; measured as time-to-first-value + save rate, never top-of-funnel signup rate.

Locked decisions

#DecisionChoice (and why)
1MirrorTappable — ✓ / ✗ + correction on every statement. First taste-model event; correcting her must have a visible payoff (the job list re-ranks).
2Insight boldnessBold with receipts when evidence allows (“~$250k — I compared 12 postings; estimate, not a promise”); sharp-but-safer fallback on thin data. Never certainty beyond evidence.
3What we askOne input (LinkedIn / CV file / free text) + skippable one-line target question. Never required.
4Motivating detail, not forcing itEach input option carries its own “why” line · sharpness meter fills as they give more · her target-guess is a correctable mirror statement · matches carry “ranked against my guess — tell me the real one” re-rank nudge.
5SignupAfter value, framed as save. Save screen leads with a real taste line (one resume bullet retold for the top match) + 3 plain what-starts-now cards + honest “this disappears unless saved.”
6Reveal pacingCombined screen, two-beat fill: her read appears first, jobs stream in beside it (“still comparing…”). Honest timing, one screen. Stacks on mobile, read on top.
7TickerOnly real steps, in plain English — every line must pass “would a non-tech friend understand this?” No jargon, no fake theater.
8First feed minuteOption A: RO narrates what she’s doing next → goal card (“everything I do gets sharper” · the seam to the Goal-setup feature) → saved matches → “what happens while you’re away.”
9Explore arrivalIn scope. RO opens by continuing their conversation; full self-explaining input block (context-aware why-lines); the role they came from is guaranteed a verdict, tagged.
10Edge casesHonest scripted recoveries, per ro-voice.html: thin input (ask for exactly two things) · junk (“no judgment, I can’t read you from it” — never matches) · weak pool (mirror + insight still delivered; “nothing’s a strong fit — widen or hold the bar? I’ll keep watch”).
11Returning users(a) saved work → straight to feed, never re-onboarded · (b) signed in, no data → “signing in told me who you are, not what you’ve built” · (c) anon repeat → remembered LinkedIn URL + “re-pull?”
12SuccessTime-to-first-value + post-matches save rate. Explicitly NOT signup rate at entry.

Data & personalization requirements (confirmed 2026-07-04)

RuleMeaning for the build
Capture every actionEvery user action across the journey is a decision_event (append-only) feeding the taste model — onboarding included. Personalization compounds across the user’s whole life with RO. Per-user only (RLS); exportable & deletable; anonymized aggregate learning only at product level.
Pre-save privacyAnonymous session stores nothing server-side — that’s the honest “disappears unless saved” claim. Non-negotiable.
Save-payload completeness ⚑ newThe signup handoff must carry ALL pre-save actions — every ✓/✗, every correction, the target answer, the re-rank request — and write them as decision events the moment the account exists. The taste model’s first entries are the corrections made before signup. (Today’s app carries profile/mirror/matches only — this is a required extension.)

Flags for the PRD (human/business calls)

Taste-line costOne extra model call per anonymous user who reaches the save screen. Recommended: pay it (strongest conversion moment). Alternative: canned example (cheaper, visibly not theirs).
Re-rank latencyPrototype shows ~1s re-rank; production is a real model call (5–15s). Decide: instant acknowledge + quiet re-sort, or block with progress. Design leans instant-acknowledge.
Insight anchorMoney ($250k) vs trajectory as the default insight class. Currently money (most falsifiable, most arresting). Open to challenge.
Anon LinkedIn pullsReal cost per pull (Apify). Rate limits exist (H3); the free-pull business policy is the human’s call.
